## 3. Building the Core Health Check Script

Now, let's construct the script. We will build a modular approach using functions for each check type. This makes the script maintainable and easy to extend later.

### Step 1: Establishing Connectivity
The foundation of any automation script is a reliable connection. In vSphere 8, we can add robust error handling here. If the connection fails or permissions are denied, the script should exit gracefully with a clear message rather than throwing a cryptic stack trace.

```powershell
function Connect-VSphere {
    param(
        [string]$Server,
        [string]$User,
        [string]$Password
    )

    try {
        $Pass = ConvertTo-SecureString $Password -AsPlainText -Force
        Connect-VIServer -Server $Server -User $User -Password $Pass -ErrorAction Stop
        Write-Host "Successfully connected to vCenter: $Server" -ForegroundColor Green
        return $true
    }
    catch {
        Write-Host "Failed to connect to vCenter. Error: $_" -ForegroundColor Red
        return $false
    }
}
```

### Step 2: Inventory Discovery
Once connected, we need to identify our targets. Instead of hardcoding a list of host names, we will dynamically retrieve all ESXi hosts. We can apply filters here based on tags or cluster names to ensure we are only checking the intended environment.

# Define the inventory scope

$Hosts = Get-VMHost -Location "Production"

### Step 3: Defining Check Functions

We will break down the health checks into logical functions. Each function returns a status object that indicates Pass, Fail, or Warning.

#### Function A: Host Connectivity & Status
This function verifies that hosts are connected to vCenter and are not unexpectedly in Maintenance Mode.

```powershell
function Test-HostStatus {
    param([VMHost]$Host)
    
    $status = [PSCustomObject]@{
        Property = "Connectivity"
        HostName = $Host.Name
        Value = $Host.ConnectionState
        Status = if ($Host.ConnectionState -eq 'Connected') { 'Pass' } else { 'Fail' }
        Message = "Host is $($Host.ConnectionState)"
    }

    # Check for unexpected states
    if ($Host.PowerState -eq 'PoweredOff') {
        $status.Message += " | WARNING: Host is Powered Off"
        $status.Status = 'Warning'
    } elseif ($Host.IsInMaintenanceMode) {
        $status.Message += " | WARNING: Unexpectedly in Maintenance Mode"
        $status.Status = 'Warning'
    }

    return $status
}
```

#### Function B: Resource Saturation
This is critical for performance tuning. We will check CPU Ready time and Memory Ballooning. In vSphere 8, accessing these metrics via the `Get-VMHost` object properties is streamlined, reducing API calls compared to legacy cmdlets.

```powershell
function Test-ResourceSaturation {
    param([VMHost]$Host)

    # Access direct host properties for efficiency in vSphere 8
    $cpuReady = Get-VmHostCpuUsage -Host $Host | Select-Object AverageReadyTimePercent
    
    # Check Cluster Memory Ballooning (requires cluster context)
    $cluster = $Host.Parent
    $memBallooning = if ($cluster) { 
        (Get-ClusterMemoryInfo -Cluster $cluster).BalloonedMem 
    } else { 0 }

    $results = [PSCustomObject]@{
        Property = "CPU Ready %"
        HostName = $Host.Name
        Value = if ($cpuReady.AverageReadyTimePercent) { $cpuReady.AverageReadyTimePercent } else { 'N/A' }
        
        # Thresholds: <5% Pass, 5-10% Warning, >10% Fail
        Status = if ($cpuReady.AverageReadyTimePercent -lt 5) { 'Pass' } 
                elseif ($cpuReady.AverageReadyTimePercent -lt 10) { 'Warning' } 
                else { 'Fail' }
    }

    # Add Memory Ballooning check if applicable
    if ($memBallooning -gt 0) {
        $results | Add-Member -NotePropertyName "MemoryBalloon" -NotePropertyValue ($memBallooning / 1GB).ToString('F2') + " GB" -Force
        # Optional: Treat ballooning as a warning condition
        if ($memBallooning -gt 500) { 
            $results.Status = 'Fail' 
            $results.Message += " | CRITICAL: High memory ballooning detected"
        } elseif ($memBallooning -gt 100) {
            $results.Status = 'Warning'
            $results.Message += " | INFO: Some memory ballooning detected"
        }
    } else {
        $results | Add-Member -NotePropertyName "MemoryBalloon" -NotePropertyValue "0.00 GB" -Force
    }
    
    return $results
}
```

#### Function C: Storage Health
Identifying disks in degraded states or latency spikes is vital. We will check the datastore health state and look for devices marked as 'degraded'.

```powershell
function Test-StorageHealth {
    param([VMHost]$Host)

    $storageResults = @()
    
    # Get datastores accessible by this host
    $datastores = Get-Datastore -Location $Host
    
    foreach ($ds in $datastores) {
        $healthCheck = [PSCustomObject]@{
            Property = "Datastore"
            HostName = $Host.Name
            DatastoreName = $ds.Name
            State = $ds.State # e.g., 'green', 'yellow', 'red'
            Status = if ($ds.State -eq 'green') { 'Pass' } elseif ($ds.State -eq 'yellow') { 'Warning' } else { 'Fail' }
        }
        
        # Check for degraded devices on this datastore
        $devices = Get-ScsiLun -Storage $ds | Where-Object { $_.DeviceState -eq 'degraded' }
        
        if ($devices) {
            $healthCheck.Message = "WARNING: Degraded device found on storage array"
            $healthCheck.Status = 'Fail' # Override status on critical storage issue
        } else {
             $healthCheck.Message = "Storage devices healthy"
        }
        
        $storageResults += $healthCheck
    }
    
    return $storageResults
}
```

#### Function D: Certificate Expiry
SSL certificate issues can cause silent failures in API calls and VM traffic. We will scan the host certificates to find those expiring within 30 days.

```powershell
function Test-CertificateExpiry {
    param([VMHost]$Host)
    
    $certs = Get-VMHostCertificate -Host $Host
    
    foreach ($cert in $certs) {
        # Correct calculation: Days remaining until expiration
        $daysRemaining = $cert.ExpirationDate - [DateTime]::Now
        
        $expiryResult = [PSCustomObject]@{
            Property = "Certificate"
            HostName = $Host.Name
            Subject = $cert.Subject
            ExpiryDate = $cert.ExpirationDate
            DaysRemaining = [Math]::Round($daysRemaining.TotalDays, 1)
            
            # Check if less than 30 days away
            Status = if ($daysRemaining.TotalDays -lt 30) { 'Fail' } else { 'Pass' }
        }
        
        if ($expiryResult.Status -eq 'Fail') {
            $expiryResult.Message = "EXPIRES SOON: In $($expiryResult.DaysRemaining) days"
        } else {
            $expiryResult.Message = "Certificate is valid"
        }
    }
    
    # If no certs found, return a Pass result to avoid null errors in the loop
    if ($certs.Count -eq 0) {
        return [PSCustomObject]@{ Property="Certificate"; HostName=$Host.Name; Value="No Certificates"; Status="Pass"; Message="No certificates configured" }
    }

    return $expiryResult
}
```

### Step 4: The Main Execution Loop
Now, we tie it all together. We iterate through the discovered hosts, running each function, and aggregate the results into a clean report.

**Important:** In vSphere 8, querying hundreds of hosts simultaneously can trigger API throttling errors. We include `Start-Sleep` to give the API time to breathe.

```powershell
$Report = @()
$SleepInterval = 2 # Seconds between host checks

foreach ($Host in $Hosts) {
    Write-Host "Processing Host: $($Host.Name)" -ForegroundColor Cyan
    
    try {
        # Run checks
        $checks = @(
            Test-HostStatus -Host $Host,
            Test-ResourceSaturation -Host $Host,
            Test-StorageHealth -Host $Host
        )
        
        # Add Certificate check
        $certCheck = Test-CertificateExpiry -Host $Host
        
        # Append to report
        foreach ($c in $checks + $certCheck) {
            $Report += $c
        }
    }
    catch {
        Write-Host "Error processing $($Host.Name): $_" -ForegroundColor Red
        # Add a generic error entry so the loop continues
        $errorEntry = [PSCustomObject]@{ Property="General"; HostName=$Host.Name; Value="Script Error"; Status="Fail"; Message=$_ }
        $Report += $errorEntry
    }

    # Prevent API Throttling
    Start-Sleep -Seconds $SleepInterval
}

# Generate Summary Table
$Summary = $Report | Group-Object Status | Select-Object Name, Count, @{Name='Status';Expression={[string]::Join(', ', $_.Group.Status)}}

$Report | Format-Table -AutoSize
```

## 4. Common Issues & Fixes

Even well-written scripts encounter roadblocks in production environments. Here are some common issues you might face and how to resolve them.

### Issue 1: "Cannot find an appropriate HostSystem"
**Cause:** The script is running from a user context where the vCenter cannot see all hosts, or the inventory has not refreshed.
**Fix:** Ensure you run `Get-View` or refresh the inventory using `Get-VMHost -Server $vcenter`. Also, verify that the service account has the "Read All Virtual Machines" and "Read All Datastores" privileges.

### Issue 2: Script Execution Policy Restrictions
**Cause:** On Windows workstations, PowerShell often blocks script execution by default (`RemoteSigned` or `Restricted`).
**Fix:** Run `Set-ExecutionPolicy RemoteSigned -Scope CurrentUser` to allow local scripts to run while maintaining security for downloaded ones.

### Issue 3: Memory Pressure During Bulk Checks
**Cause:** Fetching massive amounts of data (like history metrics) on hundreds of hosts can spike the vCenter API server memory or cause throttling errors.
**Fix:** 
1. Add a delay between host iterations using `Start-Sleep -Seconds 2` inside the loop (as shown above).
2. Avoid requesting long history windows in your queries unless necessary.

### Issue 4: Certificate Parsing Errors
**Cause:** Some certificates use elliptic curve algorithms that older .NET versions struggle with, or certificate chain validation fails due to intermediate CA issues.
**Fix:** Ensure you are using PowerShell 5.1 or Core 6+ and update the vCenter server patches to ensure full compatibility with modern certificate standards in vSphere 8.

### Issue 5: "Get-ScsiLun" Command Not Found
**Cause:** In some vSphere 8 updates, `Get-ScsiLun` behavior changed regarding storage backends.
**Fix:** If you receive errors accessing Lun info on NFS or specific VMFS versions, update your PowerCLI module to the latest version compatible with vSphere 8:
```powershell
Update-Module -Name VMware.VimAutomation.Core
```
